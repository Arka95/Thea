## Part 1: Agent Skills

### What Are Skills?

**Skills** are specialized workflows that agents auto-load based on task relevance. Each skill is a structured guide with step-by-step instructions.

**Format**: Each skill is a folder with `SKILL.md` file containing YAML frontmatter + markdown instructions.

### Official Skills Directory

```
.vscode/skills/
├── document-generator/
│   └── SKILL.md             ← Generate/update docs from git + chat history
├── performance-profiler/
│   └── SKILL.md             ← Profile GPU/CPU/memory usage
├── video-pipeline-architect/
│   └── SKILL.md             ← Make architectural decisions
├── config-validator/
│   └── SKILL.md             ← Validate config files
└── dependency-analyzer/
    └── SKILL.md             ← Analyze dependencies & vulnerabilities
```

### How Agents Use Skills

1. **Auto-Discovery**: VS Code agents automatically find skills in `.vscode/skills/`
2. **Context-Aware Loading**: Agent reads skill title/description, loads when relevant
3. **Manual Invocation**: User can explicitly request skill: "Use document-generator to create API reference"
4. **Workflow Following**: Agent reads SKILL.md and follows step-by-step instructions

### SKILL.md Format (Required)

Every skill must have this structure:

```yaml
---
name: skill-name-kebab-case
description: One-line description of what the skill does. Use when [context].
user-invocable: true
disable-model-invocation: false
---

# Skill Title

## When to Use This Skill
- Specific use case 1
- Specific use case 2

## Workflow/Process
Step-by-step instructions...

## Key Considerations
Important details...

## Integration
How it works with other components...
```

## Part 2: Tasks (Runnable Workflows)

### What Are Tasks?

**Tasks** are executable workflows defined in `.vscode/tasks.json`. They can be:
- Run via VS Code UI (`Terminal > Run Task`)
- Run via keyboard shortcut (`Ctrl+Shift+B` for default)
- Invoked by external agents via terminal commands
- Integrated into CI/CD pipelines

### Official Tasks (7 total)

| Task | Command | Purpose | Default Group |
|------|---------|---------|---|
| Run Fast Tests | `pytest tests/fast -v` | Quick unit test validation | test |
| Run All Tests | `pytest tests --tb=short` | Comprehensive test suite | test |
| Lint with Ruff | `ruff check . --fix` | Check + auto-fix code issues | build |
| Format with Black | `black . --line-length 100` | Format code consistently | build |
| Type Check | `mypy . --ignore-missing-imports` | Validate type annotations | build |
| Security Audit | `bandit -r . -ll` | Scan for security issues | build |
| Run Main Pipeline | `python main.py` | Execute video processing pipeline | build |

### tasks.json Structure

```json
{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "Run Fast Tests",
      "type": "shell",
      "command": "pytest",
      "args": ["tests/fast", "-v"],
      "group": {
        "kind": "test",
        "isDefault": true
      },
      "presentation": {
        "reveal": "always",
        "panel": "new"
      },
      "problemMatcher": ["$pytest"]
    }
  ]
}
```

### How Agents Use Tasks

```powershell
# Agents run tasks via terminal
cd d:\Projects\Thea
pytest tests/fast -v          # From "Run Fast Tests" task

ruff check . --fix            # From "Lint with Ruff" task
black . --line-length 100     # From "Format with Black" task
python main.py                # From "Run Main Pipeline" task
```

### How to Add a New Task

1. **Edit `.vscode/tasks.json`**:
   ```json
   {
     "label": "Your Task Name",
     "type": "shell",
     "command": "python",
     "args": ["-m", "thea.your_module"],
     "group": "build",
     "presentation": {
       "reveal": "always"
     }
   }
   ```

2. **Required Fields**:
   - `label` - Display name for the task
   - `type` - Usually "shell" for running commands
   - `command` - The executable (python, pytest, etc.)
   - `args` - Arguments to pass

3. **Optional Fields**:
   - `group` - Category (test, build) + isDefault flag
   - `presentation` - How to show output
   - `problemMatcher` - Parse output for errors
   - `dependsOn` - Run other tasks first

4. **Test the task**:
   - `Ctrl+Shift+B` to run (if `isDefault: true`)
   - Or `Terminal > Run Task` to see all tasks
   - Verify output appears in terminal

5. **Document in this file** (update the tasks table above)

## Part 3: Debug Configurations

### What Are Launch Configurations?

**Launch configurations** in `.vscode/launch.json` let agents start debuggers and inspect code execution.

### Official Debug Configurations (5 total)

| Config | Purpose | Console | Debug Mode |
|--------|---------|---------|---|
| Python: Main Pipeline | Debug main.py | integrated | Full (justMyCode: false) |
| Python: Current File | Debug active file | integrated | Fast (justMyCode: true) |
| Python: Fast Tests | Debug tests/fast | integrated | Test mode |
| Python: Slow Tests | Debug tests/slow with output | integrated | Test mode |
| Python: Profile Memory | Run memory_profiler | integrated | Profiling |

### launch.json Structure

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Python: Main Pipeline",
      "type": "python",
      "request": "launch",
      "program": "${workspaceFolder}/main.py",
      "console": "integratedTerminal",
      "justMyCode": false,
      "showReturnValue": true
    }
  ]
}
```

### How to Add a New Debug Configuration

1. **Edit `.vscode/launch.json`**:
   ```json
   {
     "name": "Your Config Name",
     "type": "python",
     "request": "launch",
     "program": "${workspaceFolder}/path/to/file.py",
     "console": "integratedTerminal"
   }
   ```

2. **Key Properties**:
   - `name` - Display name
   - `type` - Language (python, node, etc.)
   - `request` - "launch" or "attach"
   - `program` - Main file to debug
   - `args` - Command-line arguments
   - `env` - Environment variables
   - `justMyCode` - Skip library code (true = faster)

3. **Test the configuration**:
   - Click `Run and Debug` panel
   - Select your config
   - Set breakpoints and step through code

## Part 4: User-Invocable Prompts

### What Are Prompts?

**Prompts** are slash commands (`/`) that users can invoke in chat to apply specialized AI behavior.

### Official Prompts (7 total)

**Located in**: `.vscode/prompts/` and `c:\Users\[user]\AppData\Roaming\Code\User\prompts\`

| Prompt | Purpose | Format |
|--------|---------|--------|
| `/code-review` | Senior code review (bugs, security, performance) | Markdown with YAML frontmatter |
| `/git-commit` | Generate meaningful commit messages | Check git status → write message |
| `/knowledge-graph-builder` | Build entity-relation architecture models | Scan structure → create entities → create relations |
| `/rubber-duck` | Debug critique mode | Find logic errors, edge cases, vulnerabilities |
| `/sequential-solver` | Structured multi-step problem solving | Restate → constraints → approaches → evaluate → solve |
| `/test-writer` | Generate test cases following patterns | Happy path → edge cases → error cases → integration |
| `/web-research` | Research information from web sources | Playwright for interactive browsing |

### How Agents Use Prompts

User invokes:
```
/code-review
```

Agent receives the prompt instructions and applies them to current context (open file, chat history, etc.).

### How to Add a New Prompt

1. **Create file** in `.vscode/prompts/`:
   ```
   .vscode/prompts/your-prompt-name.md
   ```

2. **Add YAML frontmatter**:
   ```yaml
   ---
   title: "Prompt Title"
   description: "One-line description of what the prompt does"
   ---
   
   # Prompt Content
   
   Your detailed instructions here...
   ```

3. **Also add to global location** (for user reuse):
   ```
   c:\Users\[user]\AppData\Roaming\Code\User\prompts\your-prompt-name.md
   ```

4. **Document in this file** (update prompts section)

## Part 5: Settings and Configuration

### settings.json Structure

```json
{
  "chat.agentSkillsLocations": [
    "${workspaceFolder}/.vscode/skills",
    "${workspaceFolder}/.github/skills"
  ],
  "mcp.servers": { ... },
  "[python]": { ... },
  "python.linting": { ... },
  "python.testing": { ... },
  "python.analysis": { ... },
  "files.exclude": { ... }
}
```

**Key Sections**:
- `chat.agentSkillsLocations` - Where to find skills
- `mcp.servers` - Model Context Protocol servers (filesystem, git, memory, seq-thinking)
- `[python]` - Python editor settings (formatter, rulers, actions)
- `python.*` - Python-specific linting, testing, analysis
- `files.exclude` - Hide generated/cache files

### How to Modify Settings

1. **Edit `.vscode/settings.json`** directly
2. **Affects all agents** working in this workspace
3. **Takes precedence** over user global settings
4. **Recommended changes**:
   - Add Python linting rules
   - Configure formatters
   - Set testing framework options
   - Change analysis strictness
